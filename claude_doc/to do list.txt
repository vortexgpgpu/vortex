# Vortex DTCU / TMA To-Do List

## P0 — Verify source state before coding

1. Confirm actual source layout under `claude_doc/`.
   - Locate Vortex tree.
   - Locate Virgo source and notes.
   - Confirm whether a separate `DTCU Note.zip` exists.

2. Inspect all DTCU source files.
   - `kernel/include/vx_tensor.h`
   - `sim/simx/d_tensor_core.h`
   - `sim/simx/d_tensor_core.cpp`
   - `sim/simx/cluster.h`
   - `sim/simx/cluster.cpp`
   - `sim/simx/constants.h`
   - `sim/simx/types.h`
   - `sim/simx/decode.cpp`
   - `sim/simx/execute.cpp`
   - `sim/simx/func_unit.h`
   - `sim/simx/func_unit.cpp`
   - `sim/simx/core.cpp`
   - `tests/regression/dtcu_basic/*`
   - `tests/regression/dtcu_compare/*`

3. Verify DTCU build flags.
   - Confirm `EXT_TCU_ENABLE` path.
   - Confirm whether any extra flag is needed for DTCU beyond TCU.
   - Confirm if `EXT_DMA_ENABLE` is currently used anywhere relevant.

4. Fix or replace stale `dtcu_basic`.
   - Current source may use old descriptor fields and too-small M/N/K.
   - Make it match current DTCU constraints:
     - M multiple of 64.
     - N multiple of `shape_n_size * 16`.
     - `shape_n_size != 0`.
     - `shape_policy == 0`.
     - output format fp32.
   - Acceptance: host CPU reference equals DTCU output; no hang.

5. Resolve `dtcu_compare` mode mismatch.
   - Source appears to use `mode=0` for in-core TCU and `mode=1` for DTCU.
   - Notes say the opposite.
   - Update comments/docs so the mapping is unambiguous.

6. Add DTCU tests to regression/CI script if missing.
   - Start with only known-passing configs.
   - Keep config explicit.

7. Run baseline DTCU tests.
   - `dtcu_basic` or replacement.
   - `dtcu_compare`.
   - Save logs under a clear directory.

## P1 — Decide TMA scope and ABI

8. Decide first TMA target.
   - Preferred first target: SimX-only.
   - Defer RTL unless explicitly requested.

9. Decide TMA staging target.
   - Option A: DTCU-private staging buffers. Recommended first.
   - Option B: existing Vortex local/shared memory. More Virgo-like, more invasive.
   - Option C: new cluster shared memory. Clean but large scope.

10. Define TMA descriptor ABI.
    - Include src/dst base addresses.
    - Include dimensions and strides.
    - Include element size / format.
    - Include direction and flags.
    - Add reserved/version fields if possible.

11. Define TMA kernel API.
    - Minimal custom instruction path:
      - `dtma_start(desc_addr)`
      - `dtma_poll()`
    - Later possible API:
      - `dtma_load(...)`
      - `dtma_store(...)`
      - `dtma_fence()`

12. Decide whether TMA should share the DTCU L2 port or get a separate L2 port.
    - Sharing is simpler.
    - Separate port may be cleaner for counters and concurrency.
    - Verify existing L2 request port constraints first.

## P2 — Implement SimX TMA

13. Add TMA type/state machine.
    - Either new `DTensorTMA`/`DTensorDMA` class or clean submodule inside `DTensorCore`.
    - States should cover descriptor request/wait, data request/wait, functional copy, done.

14. Add L2-facing request generation.
    - Build cache-line request lists from descriptor dimensions and strides.
    - Model read and write direction correctly.
    - Avoid only issuing one representative request if the goal is TMA traffic fidelity.

15. Add functional copy.
    - Use RAM read/write after timing requests complete.
    - Preserve layout and stride exactly.
    - Add debug prints behind a trace/debug flag, not always-on spam.

16. Add busy/done/error behavior.
    - Define behavior for start while busy.
    - Define behavior for invalid descriptors.
    - Make poll semantics explicit.

17. Add kernel intrinsics.
    - Add TMA start/poll intrinsics in `kernel/include/vx_tensor.h` or a new header.
    - Keep custom instruction encoding documented.

18. Add decode/execute integration.
    - Add new `TcuType` entries.
    - Add decode cases.
    - Add functional-unit routing if needed.
    - Add execute calls into cluster/TMA object.

## P3 — Add tests for TMA

19. Add `dtma_basic`.
    - Host initializes source buffer.
    - Kernel starts TMA copy and polls.
    - Host verifies destination buffer byte-exactly.

20. Add `dtma_strided`.
    - Test non-contiguous rows/leading dimensions.
    - Catch layout/stride bugs early.

21. Add `dtcu_tma_basic`.
    - TMA loads A/B tiles.
    - DTCU computes.
    - TMA stores D.
    - Host compares D with CPU reference.

22. Add `dtcu_tma_compare`.
    - Compare in-core TCU, direct DTCU, and TMA+DTCU.
    - Print cycles, instructions, L2 reads/writes, mem reads/writes.

23. Add negative tests.
    - Invalid descriptor.
    - Unsupported format.
    - Partial tile if unsupported.
    - Busy start.

## P4 — Improve DTCU/TMA memory model fidelity

24. Separate descriptor traffic, operand traffic, and output traffic in counters/logs if possible.

25. Add TMA byte/request counters.
    - bytes read
    - bytes written
    - cache lines read
    - cache lines written

26. Replace overly representative traffic with cache-line accurate request lists where feasible.

27. Verify L2 counter direction.
    - TMA path should show DTCU/TMA-originated memory traffic.
    - In-core TCU path should show core-originated load/store traffic.

## P5 — Optional RTL path after SimX is stable

29. Decide RTL TMA placement.
    - DTCU submodule.

30. Add RTL port wiring only after design is clear.

## Current known risks

- `dtcu_basic` may be stale relative to current `DTensorCore` tile rules.
- Notes and source disagree on `dtcu_compare` mode mapping.
- Current DTCU appears SimX-only; RTL completeness is not confirmed.
- Functional RAM access and L2 timing requests are currently split, so memory counters may not be fully faithful.
- TMA target memory is not yet decided.
- Partial tile support is not present in current DTCU.

## First acceptance checkpoint

The first meaningful checkpoint is:

- Current DTCU tests are repaired and passing.
- Source/docs agree on mode mapping and descriptor fields.
- `dtma_basic` passes byte-exact copy.
- `dtcu_tma_basic` passes CPU-reference correctness.
- Logs show TMA-originated L2/memory traffic.
